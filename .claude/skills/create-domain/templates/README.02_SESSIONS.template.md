# Sessions

Feedback and memory system. Captures session discussions, changes, and decisions. Users can revisit or inject session context into future conversations.

## Purpose

This folder stores chronological logs of interactions, decisions, and changes made during work sessions. It serves as the domain's memory system, allowing continuity across conversations.

## File Naming

Files must use `YYYY-MM-DD_title.md` format.

**Examples:**
- `2026-01-15_sync.md`
- `2026-02-10_planning_review.md`
- `2026-02-11_feature_discussion.md`
- `2026-02-11_bug_triage.md`

## Usage

- Create one log per session or significant interaction
- Include decisions made and their rationale
- Reference these logs to maintain continuity across sessions
- Inject session context into new conversations when needed

## What to Include

- Session date and participants
- Topics discussed
- Decisions made (with reasoning)
- Action items and outcomes
- Changes to domain or projects
- Open questions for follow-up
